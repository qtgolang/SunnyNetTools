REST 调用（invoke）

请求体为单行 JSON，字段 op + args：

POST /sunnynet/tools/invoke
Content-Type: application/json

{
  "id": 1,
  "op": "get_status",
  "args": {}
}

响应：

{
  "id": 1,
  "ok": true,
  "result": { "sunnyRunning": true, "mainCount": 12, ... }
}

失败时 ok: false，error 为错误说明。

Streamable MCP（Cursor 等）

在 MCP 客户端中配置 Server URL：

http://127.0.0.1:6987/sunnynet/tools/mcp

工具名前缀为 sunnynet_，例如 sunnynet_get_status、sunnynet_main_slice，与 REST 的 op 一一对应（去掉前缀即可）。

ID 约定（listIndex / theology / rowId）

  • 界面「序号」列 = listIndex（从 1 开始，第 66 条 → listIndex: 66 或 main_slice 的 offset: 65）。
  • theology = 内部会话主键（整数，可为负）。
  • rowId = theology 的十进制字符串，恒满足 rowId === String(theology)（例如 theology 66 → rowId "66"）。不是 http-* 别名，也不是 listIndex。
  • 多行操作一次传入 listIndexes、rowIds、ids 或 theologies，禁止循环单条调用。
  • 流表 streamKey 一般为 rowId（即 theology 字符串）；子消息用 messageId（整数）。
批量约定：写注释用 main_row_note_set；读断点状态用 main_row_break_get（breakMode 0/1/2）；断点放行用 break_continue / break_continue_all；读大正文用 http_get_part / stream_get_part 分块。
